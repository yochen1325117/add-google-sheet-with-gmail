# Google 表單與試算表設定（讓使用者可提交）

投票結果是 **POST 到 Google 表單**，表單的回應可 **自動寫入 Google 試算表**。因此需要：**1 個 Google 表單 + 與試算表連結**。

---

## 一、你需要準備的兩件事

| 項目 | 說明 |
|------|------|
| **Google 表單 ID** | 表單網址中間那串，例如 `1FAIpQLSe...` |
| **每個題目的 entry id** | 每題對應一個 `entry.數字`，例如 `entry.123456` |

表單建立好並**連結到試算表**後，每次有人透過投票網站送出，就會多一筆表單回應，試算表會多一列。

---

## 二、步驟 1：建立 Google 表單並對應題目

1. 前往 [Google 表單](https://forms.google.com/) → **建立空白表單**（或從試算表：**擴充功能** → **Google 表單** → **建立表單**，可選「連結到現有試算表」）。
2. 在表單中**依序新增以下題目**（題型用「簡答」或「段落」即可，方便對應試算表欄位）：

| 順序 | 題目標題（自訂） | 本專案對應欄位 | 說明 |
|------|------------------|----------------|------|
| 1 | Email | email | 必填，投票者 email |
| 2 | 員工編號 | employeeId | 必填 |
| 3 | 提交時間 | submittedAt | 自動帶入 ISO 時間 |
| 4 | 全能辦公桌改造王-第1名 | award1_1 | 流水號 1～200 或空白 |
| 5 | 全能辦公桌改造王-第2名 | award1_2 | |
| 6 | 全能辦公桌改造王-第3名 | award1_3 | |
| 7 | 北歐極簡主義擁護者-第1名 | award2_1 | |
| 8 | 北歐極簡主義擁護者-第2名 | award2_2 | |
| 9 | 北歐極簡主義擁護者-第3名 | award2_3 | |
| 10 | 空間利用的收納魔術師-第1名 | award3_1 | |
| 11 | 空間利用的收納魔術師-第2名 | award3_2 | |
| 12 | 空間利用的收納魔術師-第3名 | award3_3 | |
| 13 | 台幣戰士-第1名 | award4_1 | |
| 14 | 台幣戰士-第2名 | award4_2 | |
| 15 | 台幣戰士-第3名 | award4_3 | |
| 16 | 五感療癒的支配者-第1名 | award5_1 | |

共 **21 題**（Email、員工編號、提交時間 + 18 個獎項名次）。題目標題可自訂，只要之後對應到正確的 entry id 即可。

---

## 三、步驟 2：表單連結到試算表

- 若表單是從試算表建立（**擴充功能** → **Google 表單** → **建立表單**），建立時可選「連結到這份試算表」。
- 若表單是單獨建立的：表單編輯頁 → **回覆** → **連結試算表** → 選擇「現有試算表」或「建立新的試算表」。

之後每一筆送出都會在試算表新增一列。

---

## 四、步驟 3：取得 Form ID

1. 在表單編輯頁點 **傳送**。
2. 點 **連結** 圖示，複製網址。  
   格式為：`https://docs.google.com/forms/d/e/**FORM_ID**/viewform`
3. 中間的 **FORM_ID**（例如 `1FAIpQLSexxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）就是表單 ID。  
   → 填到專案 `src/config.ts` 的 `FORM.id`。

---

## 五、步驟 4：取得每題的 entry id

每題在表單裡都對應一個 `entry.數字`，要用來填 `src/config.ts` 的 `FORM.entry`。

### 方法 A：預填連結（建議）

1. 表單中每一題**隨便填一個值**（例如 Email 填 `a@a.com`、員工編號填 `1`、提交時間填 `2025-01-01`、各獎項填 `1`）。
2. 點 **傳送** → **連結** 右側 **⋮** → **取得預填連結**。
3. 複製產生的網址，會長得像：  
   `https://docs.google.com/forms/d/e/FORM_ID/viewform?usp=pp_url&entry.123456=a@a.com&entry.234567=1&entry.345678=2025-01-01&...`
4. 網址中 **`entry.數字`** 就是該題的 id。  
   依你表單的**題目順序**對照上面的「本專案對應欄位」表格，把每個 `entry.數字` 填進 `config.ts` 的對應 key。

### 方法 B：檢視網頁原始碼

1. 點表單的 **預覽**（眼睛圖示），在**新分頁**打開表單。
2. 在預覽頁 **右鍵 → 檢視網頁原始碼**（或 Ctrl+U / Cmd+Option+U）。
3. 搜尋 **`entry.`**，會看到很多 `name="entry.xxxxx"`。
4. 從上到下對應表單題目順序，第 1 題對應第一個 entry、第 2 題對應第二個…，依此類推對到 `config.ts` 的欄位。

---

## 六、步驟 5：填寫專案設定

打開 **`src/config.ts`**，修改兩處：

### 1. 表單 ID

```ts
export const FORM = {
  id: "這裡貼上你的_FORM_ID",   // 例如 1FAIpQLSe...
  // ...
};
```

### 2. 各欄位 entry 對應

把你在步驟 4 查到的每個題目的 `entry.數字`，填進對應的 key（key 名稱對照上表「本專案對應欄位」）：

```ts
  entry: {
    email: "entry.你查到的數字",
    employeeId: "entry.你查到的數字",
    submittedAt: "entry.你查到的數字",
    award1_1: "entry.xxx",
    award1_2: "entry.xxx",
    award1_3: "entry.xxx",
    award2_1: "entry.xxx",
    award2_2: "entry.xxx",
    award2_3: "entry.xxx",
    award3_1: "entry.xxx",
    award3_2: "entry.xxx",
    award3_3: "entry.xxx",
    award4_1: "entry.xxx",
    award4_2: "entry.xxx",
    award4_3: "entry.xxx",
    award5_1: "entry.xxx",
    award5_2: "entry.xxx",
```

存檔後重新執行 `npm run dev` 或重新建置，使用者送出投票時就會寫入你的 Google 表單，並同步到試算表。

---

## 七、本專案實際送出的內容

| 欄位 | 送出的值 |
|------|----------|
| email | 使用者輸入的 Email |
| employeeId | 使用者輸入的員工編號 |
| submittedAt | 送出當下的時間（ISO 字串，例如 `2025-02-25T10:00:00.000Z`） |
| award1_1 ～ award6_3 | 各名次選擇的**流水號**（1～200），未選則為空字串 |

表單題型用「簡答」即可；若希望試算表欄位較整齊，可把「提交時間」設成日期時間、其餘用文字。

---

## 八、若只想用試算表、沒有表單

本專案是透過 **Google 表單的 formResponse** 寫入資料，表單再寫入試算表。  
**無法**跳過表單、直接 POST 到試算表（試算表沒有對外收 POST 的網址）。

所以流程一定是：**投票網站 → Google 表單 →（表單回應）→ 試算表**。  
你已開好的試算表，只要在建立表單時選「連結到現有試算表」即可把表單回應存到同一份試算表。

---

## 九、疑難排解：送出時出現 401 Unauthorized

**原因**：Google 表單被設成「只有登入 Google 帳號的人才能填寫」，而投票網站是匿名 POST，沒有帶登入資訊，所以被拒絕。

**處理方式**：

1. 打開你的 **Google 表單**（編輯模式）。
2. 點右上角 **「設定」**（齒輪圖示）。
3. 切到 **「回覆」** 分頁。
4. 找到 **「回覆對象」** 或 **「限制為僅限登入的使用者」** 之類的選項。
5. 改為 **「任何人都可回覆」**（或取消「僅限登入的使用者」）。
6. 若有 **「收集電子郵件地址」**，可依需求開啟或關閉，不影響 401；影響 401 的是「誰可以填寫」必須是任何人。
7. 儲存後再從投票網站試送一次。
